# Catalog discovery

## Source of truth

The upstream catalog is a single JSON document served at:

```
https://www.aitmpl.com/components.json
```

It is regenerated by the upstream maintainer from the repo at
`davila7/claude-code-templates` and contains every component across all types
(agents, commands, mcps, hooks, settings, skills).

## Cache

Local cache:

```
~/.cache/aitmpl-compose/catalog.json
```

The cache is refreshed automatically when older than 24 hours.
Manual refresh:

```sh
aitmpl-compose catalog refresh
```

## Schema (per entry)

```ts
interface CatalogEntry {
  name: string;          // leaf name, e.g. "frontend-developer"
  path: string;          // upstream path, e.g. "development-team/frontend-developer.md"
  category: string;      // e.g. "development-team"
  type: ComponentType;   // agent | command | mcp | hook | setting | skill
  description?: string;
  install_name: string;  // computed: `${category}/${name}` (the value used in --workflow YAML and CLI flags)
}
```

The `install_name` is the canonical identifier used in workflow.yaml steps,
pack components, and `npx claude-code-templates` CLI flags.

## Top-level shape

```ts
interface Catalog {
  fetched_at: string;          // ISO timestamp
  source_url: string;
  components: {
    agent:   CatalogEntry[];
    command: CatalogEntry[];
    mcp:     CatalogEntry[];
    hook:    CatalogEntry[];
    setting: CatalogEntry[];
    skill:   CatalogEntry[];
  };
}
```

Upstream pluralizes the keys (`agents`, `commands`, ...). The local catalog
uses singular type names matching the workflow.yaml `type` field.

## Templates

The 14 upstream templates are not currently published in the components.json
catalog. They live as directories under `cli-tool/templates/<name>/` in the
upstream repo. For now, `aitmpl-compose compose --templates` resolves against
a small hardcoded mapping in `scripts/aitmpl-compose`. To add more, follow the
upstream README and add an entry to `KNOWN_TEMPLATES`.

Future: a `templates refresh` subcommand could parse the upstream repo's
`cli-tool/templates/*/README.md` and `.claude/` contents to materialize this
mapping live. Until then, the workflow.yaml path is the recommended route
for team-shareable bundles.

## Why fetch live?

The catalog ships ~1000 entries and changes frequently as the upstream
community adds components. A live fetch ensures any component the user
references actually exists at the version they will install. The 24-hour TTL
keeps the operation snappy in interactive use without going stale on weekly
or monthly authoring sessions.
